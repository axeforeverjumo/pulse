<task type="auto">
  <name>Task 1: Deploy OpenHands Docker service on production server</name>
  <files>
    server:/opt/openhands/docker-compose.yml
    server:/opt/openhands/.env
    server:/root/.openhands/config.toml

  <action>
SSH into 85.215.105.45 as root and execute the following steps:

1. Create directory structure:
   ```
   mkdir -p /opt/openhands
   mkdir -p /root/.openhands
   ```

2. Create `/opt/openhands/.env` with these variables (user must provide actual values):
   ```
   ANTHROPIC_API_KEY=<user-provided>
   SANDBOX_GITHUB_TOKEN=<user-provided>
   ```

3. Create `/root/.openhands/config.toml` with this content:
   ```toml
   [core]
   workspace_base = "/opt/openhands/workspace"
   runtime_startup_env_vars = { GITHUB_TOKEN = "${SANDBOX_GITHUB_TOKEN}" }

   [llm]
   provider = "anthropic"
   model = "anthropic/claude-sonnet-4-5-20250929"
   api_key_env = "ANTHROPIC_API_KEY"
   num_retries = 4
   retry_min_wait = 5
   retry_max_wait = 30
   ```

4. Create `/opt/openhands/docker-compose.yml`:
   ```yaml
   services:
     openhands:
       image: docker.openhands.dev/openhands/openhands:1.6
       container_name: openhands-app
       restart: unless-stopped
       ports:
         - "127.0.0.1:3100:3000"
       environment:
         - AGENT_SERVER_IMAGE_REPOSITORY=ghcr.io/openhands/agent-server
         - AGENT_SERVER_IMAGE_TAG=1.15.0-python
         - LOG_ALL_EVENTS=true
         - ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY}
         - SANDBOX_GITHUB_TOKEN=${SANDBOX_GITHUB_TOKEN}
       volumes:
         - /var/run/docker.sock:/var/run/docker.sock
         - /root/.openhands:/.openhands
         - /opt/openhands/workspace:/opt/workspace_base
       extra_hosts:
         - "host.docker.internal:host-gateway"
       env_file:
         - .env
   ```

5. Pull the image and start:
   ```
   cd /opt/openhands
   docker compose pull
   docker compose up -d
   ```

6. Wait 30 seconds for startup, then check container is running:
   ```
   docker compose ps
   docker compose logs --tail=50
   ```

7. Verify port binding is localhost-only (INFRA-03 security):
   ```
   ss -tlnp | grep 3100
   ```
   Must show `127.0.0.1:3100`, NOT `0.0.0.0:3100`.

8. Verify no port conflicts with existing services (3010, 4200, 5175, 8000, 8001, 8080, 8200, 9000, 9100, 18789).

IMPORTANT: The `restart: unless-stopped` policy satisfies INFRA-01 (survives reboot) as long as Docker itself is enabled at boot. Verify with: `systemctl is-enabled docker`. If not enabled, run `systemctl enable docker`.
  </action>
  <verify>
    <automated>ssh root@85.215.105.45 "docker ps --filter name=openhands-app --format '{{.Status}}' | grep -q 'Up' && ss -tlnp | grep 3100 | grep -q '127.0.0.1' && echo 'PASS: Container running, bound to localhost' || echo 'FAIL'"</automated>
  </verify>
  <done>
    - OpenHands container is running (docker ps shows "Up")
    - Port 3100 is bound to 127.0.0.1 only
    - Docker service is enabled at boot (systemctl is-enabled docker = enabled)
    - restart: unless-stopped policy is set (container survives docker restart)
    - No port conflicts with existing services
  </done>

   ```

   ```
   Should return a conversation object with an ID.

3. Restart Docker and verify container comes back:
   ```
   docker restart openhands-app

   ```
   Container must be "Up" and API must respond after restart.

**INFRA-03: Full code execution test**

4. Use the OpenHands CLI inside the container (or headless mode) to run a real task. Create a small test:

   ```
   docker exec openhands-app openhands --headless --always-approve \

   ```
   
   If the CLI approach does not work inside the container, use the REST API approach:
   - Create a conversation via POST /api/v1/app-conversations
   - Send task via the conversation WebSocket or API

NOTE: If the headless CLI does not work from inside the container (it may require a different entrypoint), fall back to using the Python SDK or REST API. The key requirement is that OpenHands can clone, edit, commit, and push -- the mechanism (CLI vs API) is secondary.
