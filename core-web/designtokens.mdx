# Core Design System (LLM Rules)

## Hard rules
1. Never invent values: use `coreDesignSystem.colors`, `spacing`, `radius`, `typography`.
2. Choose text/background colors by **semantic intent**, not by vibe.
3. Use only token spacing (`xs/sm/md/lg/xl/xxl`) and radius (`sm/md/lg/full`).
4. Typography must use the named styles in `coreDesignSystem.typography`.

## Notes
- Gradient tokens are allowed only where explicitly defined (e.g., `exercise`).
- Shadow colors include 8-digit hex with alpha.
